step 1 — know what to expect

Three easy-to-understand rounds

Nothing here is a surprise. Click each round below to see exactly what it covers, how long you get, and simple tips to do well — no experience assumed.

Round 1 — Problem Solving (35 min)

You'll get two problems that start simple and get a little trickier. There's no single "right" way in — even a working solution that isn't the fastest possible one earns you real credit.

lists & text basic patterns step-by-step logic
Tip: Talk (or write) through your thinking as you go — partial credit exists, and reviewers want to see how you reason, not just the final answer.

Round 2 — Planning It Out (35 min)

You'll be given one open-ended prompt, like "how would you build X?" You sketch out a simple plan and explain your choices out loud — there's no single correct diagram, so don't worry about getting it "perfect."

big-picture thinking organizing information explaining your choices
Tip: Start by asking clarifying questions before you draw anything. Reviewers care more about your reasoning than a polished diagram.

Round 3 — Finding the Bug (20 min)

You'll get a small, realistic project with something broken inside it. Your job is to figure out what's going wrong and fix just that part — you don't need to rewrite anything else.

reading error messages careful testing small, safe fixes
Tip: Read the error message slowly before touching any code — it almost always tells you exactly where to look first.

step 4 — pick your language

Sample questions in your language

The real assessment lets you choose from several languages. Here's one beginner-friendly sample question for each, so you know exactly what the questions look like.

Beginner strings loops

Reverse a word

Write a function called reverseString that reverses the letters in a word, in place — without using a second array to hold the result.

starter.c
#include <stdio.h>

void reverseString(char *str) {
    // your code goes here
}

int main(void) {
    char word[] = "hello";
    reverseString(word);
    printf("%s\n", word); // should print: olleh
    return 0;
}
Hint: Try two pointers — one at the start, one at the end — and swap characters as they move toward the middle.
Beginner numbers conditionals

Check for a prime number

Write a function called isPrime that returns true if a number is prime, and false otherwise.

starter.cpp
#include <iostream>
using namespace std;

bool isPrime(int n) {
    // your code goes here
}

int main() {
    cout << isPrime(7) << endl;  // should print: 1
    cout << isPrime(8) << endl;  // should print: 0
    return 0;
}
Hint: You only need to check numbers up to the square root of n — no need to test every number below it.
Beginner arrays functions

Find the largest number

Write a function called findLargest that takes an array of numbers and returns the largest one.

Hint: Keep a running "biggest so far" as you look at each number, and update it whenever you see something bigger.
